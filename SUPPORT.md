# Support

`debug-recorder-mcp` is a local-first MCP server. Please keep public support
requests focused on reproducible behavior, documentation gaps, package install
problems, and feature proposals for this repository.

## Where to Ask

- Bugs: open a bug report with the repository issue form.
- Feature requests: open a feature request with the repository issue form.
- Documentation gaps: open a documentation task with the repository issue form.
- Public security hardening: open a security hardening issue.
- Private vulnerabilities: use GitHub Security Advisories from the Security tab.

Do not paste secrets, bearer tokens, private debug logs, or full local database
contents into public issues or pull requests.

## Response Targets

These are maintainer targets, not guaranteed service-level agreements:

| Request type                                 | Target acknowledgement | Target next step                                 |
| -------------------------------------------- | ---------------------- | ------------------------------------------------ |
| Private vulnerability                        | 48 hours               | Initial assessment within 5 business days        |
| P0 release, install, CI, or security blocker | 1 business day         | Fix, workaround, or owner within 2 business days |
| P1 compatibility, product, or governance gap | 2 business days        | Plan or owner within 5 business days             |
| P2 quality, test, DX, or maintainability gap | 5 business days        | Triage decision within 10 business days          |
| P3 polish, demo, community, or roadmap item  | Best effort            | Roadmap decision when capacity allows            |

## Out of Scope

- General MCP client support that does not involve this package.
- Recovery of local SQLite databases not produced by this package.
- Private support for unpublished forks or modified distributions.
- Security reports that require sharing exploit details publicly.
